## Engine Gap Punch-List

Ranked by number of discoverable cases each gap blocks, then by estimated
implementation difficulty (easiest wins first within a tier).

### 1. `mirror_source` enum derivation â€” blocks **4** cases (tie)

All `personinfo_basic` cases skip on this.  When `mirror_source: true` is set
on an `EnumDerivation`, every permissible value in the source enum must be
copied verbatim to the target, unless overridden by an explicit
`permissible_value_derivations` entry.  The engine currently ignores this flag.

**Cases unlocked:** `golden/personinfo_basic/Container-001`,
`golden/personinfo_basic/Person-001`, `examples/personinfo_basic/Container-001`,
`examples/personinfo_basic/Person-001` â€” though these also require the
list-comprehension `expr` gap (#5) to be resolved before they can PASS.

**Location to fix:** `crates/linkml-map-core/src/engine/mod.rs` â€”
`transform_enum` / enum derivation path.

---

### 2. `asteval` `case()` built-in in `expr:` strings â€” blocks **4** cases (tie)

The `measurements` transform uses:
```yaml
expr: "case( ({height}.unit == 'cm', {height}.value), (True, NULL) )"
```
The Python `asteval` library provides `case()` as a built-in conditional;
the Rust `eval_expr_with_mapping` has no equivalent.  Once `case()` is added
to the Rust evaluator (or replaced with an `if`/`else` rewrite), all
`measurements` cases unlock.

**Cases unlocked:** `golden/measurements/PersonQuantityValue-001`,
`golden/measurements/PersonQuantityValue-002`,
`examples/measurements/PersonQuantityValue-001`,
`examples/measurements/PersonQuantityValue-002`.

**Location to fix:** `crates/linkml-map-core/src/expr/` â€” add `case` as a
named function in the evaluator's built-in dispatch table.

---

### 3. FK join / indexed object lookup â€” blocks **2** cases

The `flattening` transform accesses `subject.id`, `subject.name`, `object.id`,
`object.name` via `expr:` slots on a `Mapping` class whose `subject` and
`object` slots reference IDs (not inlined objects).  The Python engine resolves
these via an object index built with `mapper.index(source_object, target=...)`.
The Rust engine has no index / FK lookup facility.

**Cases unlocked:** `golden/flattening/MappingSet-001`,
`examples/flattening/MappingSet-001`.

**Location to fix:** Add an `ObjectIndex` map to `ObjectTransformer` and a
corresponding `index(source, root_class)` method; resolve `expr: field.nested`
through the index when `field` is an identifier-ranged slot.

---

### 4. URI/CURIE range coercion â€” blocks **2** cases

The `type_coercion` transform requests `range: uri` and `range: curie` on
certain slots.  The Python engine expands/contracts CURIEs using the schema
prefix map; the Rust engine leaves the value unchanged.

Expected output:
- `id_as_url: "https://example.org/person/1"` (from `id: "P:1"` + prefix `personinfo:`)
- `url_as_curie: "example:foo"` (from full URI + reverse prefix map)

**Cases unlocked:** `golden/type_coercion/MyRecord-001`,
`examples/type_coercion/MyRecord-001`.

**Location to fix:** `crates/linkml-map-core/src/engine/mod.rs` â€” coercion
branch; add CURIE expansion/contraction using the transform spec `prefixes`
map.

---

### 5. Python list-comprehension / multi-statement `expr:` blocks â€” blocks **4** cases (once #1 resolved)

Several `expr:` values in `personinfo_basic` contain multi-line Python:
```python
d_test = [x.important_event_date for x in src.has_important_life_events
          if str(x.event_name) == "PASSED_DRIVING_TEST"]
if len(d_test):
    target = d_test[0]
```
The Rust evaluator handles single-expression `expr:` strings but not
multi-statement blocks with assignment, list comprehensions, or `target =`.

**Cases unlocked (after #1):** all four `personinfo_basic` cases.

**Location to fix:** `crates/linkml-map-core/src/expr/` â€” either extend the
evaluator to support multi-statement blocks with `target =` semantics, or
integrate a minimal embedded scripting engine (e.g. `rune`, `rhai`).

### 6. Scalarâ†”multivalued coercion â€” blocks **1** case

The `single_value_for_multivalued` fixture has source slots that are sometimes
a single string (`greeting: namaste`) and sometimes a list
(`greeting: [konnichiwa, hello, bonjour]`).  The target slot `language` is
always multivalued.  The engine currently passes scalar values through unchanged
rather than wrapping them in a one-element list.

**Cases unlocked:** `examples/single_value_for_multivalued/Container-001`.

**Location to fix:** `crates/linkml-map-core/src/engine/mod.rs` â€” in the slot
coercion path, when the target slot is `multivalued` and the source value is a
scalar, wrap it in `Value::List(vec![scalar])`.

### 7. BioLink metamodel `deprecated: true` type mismatch â€” blocks **1** case

The `linkml_meta` crate's `SlotDefinition` struct expects `deprecated` as an
`Option<String>` but the BioLink YAML writes `deprecated: true` (a boolean).
This causes a serde parse error on schema load.

**Cases unlocked:** `examples/biolink/gene-001`.

**Location to fix:** `LinkML-MCP/rust/src/metamodel` â€” change `deprecated` to
`Option<serde_json::Value>` or add a custom deserialiser that accepts both
bool and string.
